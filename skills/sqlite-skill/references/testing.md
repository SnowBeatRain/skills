# 测试策略

SQLite 离线功能测试要覆盖“新装可用、升级可靠、离线可写、恢复可同步、失败可恢复、数据不泄露”。

## 初始化与迁移

- 新装首次启动能创建所有表和索引。
- 从每个历史 `user_version` 或 `schema_migrations` 版本升级到最新版。
- 为重要历史版本保留真实旧库 fixture，不只用内存库临时创建。
- migration 中断后再次启动能恢复或给出明确错误。
- 重复执行初始化不会破坏已有数据。
- 迁移后业务数据、outbox、cursor 不丢失。

## DAO / Repository

- insert / update / query / soft delete 行为正确。
- 所有外部输入走参数绑定。
- 动态排序字段只接受白名单。
- 事务失败时业务表和 `sync_outbox` 同时回滚。

## 离线同步

- 离线新增、编辑、删除后产生正确 outbox。
- 网络恢复后按顺序推送 pending 任务。
- API 超时、500、401、冲突响应都有明确状态。
- 重复提交同一 mutation 不产生重复服务端数据。
- 上传成功但本地更新状态前崩溃，重启后能恢复。
- 拉取远端增量与本地 pending 修改冲突时进入 `conflicted` 或按策略合并。

## 宿主环境验证

| 宿主类型 | 重点 |
|---|---|
| 移动端宿主 | 数据库路径、事务、锁、批量写入、应用强杀重启 |
| 桌面端宿主 | 文件路径、备份、并发、多进程访问 |
| 浏览器/WASM 宿主 | 持久化后端、刷新页面、存储配额、worker/主线程边界 |
| 后端/脚本宿主 | 文件权限、并发写入、备份和部署路径 |

## 安全与隐私

- 退出登录或切换账号后不能看到上个账号数据。
- 日志不包含 token、密钥、证件号、手机号、完整敏感 payload。
- 数据库文件、附件缓存、同步失败记录都有清理策略。

## 压力与异常

- 低磁盘空间。
- 大批量导入。
- 大 offset / 长列表分页。
- 多页面同时触发写入。
- 应用/工具更新、回滚、卸载重装或数据目录清理。

## 交付质量指标

可用以下指标判断 skill 产物是否比临时回答更可靠：

- 平台/宿主技术选型没有把 SQLite 与非 SQLite 存储混淆。
- schema、migration、DAO、sync、testing 至少覆盖用户请求的关键链路。
- 迁移测试覆盖新装、逐版本升级、重复启动和失败恢复。
- 同步测试覆盖 pending、syncing、synced、failed、conflicted 状态。
- 有明确的真实环境验证计划；模拟或内存库不能替代关键宿主验证。
- 增加文件或代码数量不是质量目标，关键是边界清晰、可验证、可回滚。


## 备份恢复演练

- 从备份恢复后 `quick_check` 通过。
- WAL 模式下备份文件一致，未遗漏必要数据。
- 恢复后 migration 能从备份版本升级到当前版本。
- 恢复错误账号/租户备份会被阻止。
- 损坏库或不兼容版本能给出明确错误和回退路径。
